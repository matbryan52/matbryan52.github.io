class aperture.display.figure.BokehFigure(name=None, pane_kwargs=None, **kwargs)[source]ÔÉÅ

Bases: BokehFigureBase, Bokeh

BokehFigureBase which is a standalone Panel component

This class subclasses pn.pane.Bokeh which means it is treated like a standard Panel component able to be inserted into layouts and also displayed directly in a notebook or made .servable()

As a result it is also a param.Parametrized class with default Parameters which come from Panel. Currently these parameters are not taken into account or set by the BokehFigure class, but this could be harmonized in the future with a bit of refactoring.

class aperture.display.figure_mixins.GeometryMixin[source]ÔÉÅ

Bases: object

Provides figure/frame geometry methods on BokehFigure

The terminology ‚Äòframe‚Äô refers to the canvas between the figure axes. Therefore frame_height / frame_width refer to the pixel size of the plotting area, excluding any elements like colorbars, axis labels etc.

_figure_default_size, int

The default size of the figure in pixels if no value is provided

_figure_size_limit, tuple[int, int]

The figure size (lower, upper) limit which the methods on this class will respect by up/downsizing at constant aspect ratio where possible

_figure_aspect_limit, float

the aspect ratio limit (both width/height and height/width) at which methods on this class will adjust dimensions to ensure the figure is displayable in a reasonable way

set_frame_aspect(aspect: float, frame_width: int | None = None, frame_height: int | None = None, apply_limits: bool = True, autoconfig: bool = True)[source]ÔÉÅ

Set the figure to have a given frame aspect ratio for a given fixed dimension (height or width)

Cannot specify both height and width, as this will prevent the aspect ratio being respected.

When computing the size to apply, the dimensions will be subject to the size and aspect ratio limits defined on this class, unless disabled. See the signature of set_frame_size() for information on this behaviour.

Parameters

  • aspect (float) ‚Äì The axes aspect ratio to target

  • frame_width (Optional[int], optional) ‚Äì The axis width to target, by default None

  • frame_height (Optional[int], optional) ‚Äì The axis height to target, by default None

  • apply_limits (bool, optional) ‚Äì Whether to apply figure size and aspect ratio limiting, by default True

  • autoconfig (bool, optional) ‚Äì Whether to run figure/element hooks to respond to changes in frame size, by default True

set_frame_size(*, frame_height: int, frame_width: int, apply_limits: bool = True, autoconfig: bool = True)[source]ÔÉÅ

Set the fixed figure frame size

If apply_limits is set, the dimensions will be clipped/adjusted to conform to size and aspect ratio limits according to the class properties _figure_size_limit and _figure_aspect_limit. The former defines a maximum figure dimension, which will be obeyed by scaling at constant aspect ratio. The latter defines an aspect ratio limit which will be obeyed by increasing the minimum dimension un the case of very tall or wide plots.

Will also notify elements which need to adapt to frame size changes if autoconfig is set.

Parameters

  • frame_height (int) ‚Äì The height to set

  • frame_width (int) ‚Äì The width to set

  • apply_limits (bool, optional) ‚Äì Whether to apply figure size and aspect ratio limiting, by default True

  • autoconfig (bool, optional) ‚Äì Whether to run hooks, by default True

estimate_displayed_size(*, frame_height: int | None = None, frame_width: int | None = None) → dict[str, int][source]ÔÉÅ

Tries to estimate the displayed figure size before it is rendered taking into account colorbars, axes labels, toolbar position etc.

Useful for optimizing layout of multiple figures. Will never be 100% correct as it depends on knowing browser-properties (text font scaling in particular) to correctly estimate the sizing.

Assumes normal BokehFigure behaviour, i.e. fixed frame canvas size and minimum height and width policies.

If none of frame_height or frame_width are provided, uses values from self.get_frame_size(). If one value is provided, calculates the other using the current frame aspect ratio.

# TODO support Legend objects outside the frame canvas

Parameters

  • frame_height (int, optional) ‚Äì The frame_height to use, by default None, in which case height will be calculated from other available properties. By default uses value from self.get_frame_size().

  • frame_width (int, optional) ‚Äì The frame_width to use, by default None, in which case height will be calculated from other available properties. By default uses value from self.get_frame_size().

Returns

Dictionary containing the estimated figure size. This can be compared to the output of self.get_displayed_size() after the figure has been rendered in the browser.

Return type

dict[str, int]

advise_outer_size(*, outer_height: int | None = None, outer_width: int | None = None)[source]ÔÉÅ

Tries to set the Figure frame size to provide an outer Figure size close to outer_height/outer_width.

The functionality is only approximate as we cannot know the rendered size of the figure until after it is rendered!

One possibility for improvement would be to use periodic callbacks to occasionally optimize the layout in response to screen size changes, and also to better measure the size of all the elements on the figure (colorbars etc). This would be a very complicated task to do well.

This function has some strange behaviour if both height and width are supplied, depending on the sizes of extra_height/width. Better to use just one where possible
