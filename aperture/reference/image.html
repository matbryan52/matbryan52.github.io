classmethod from_numpy(*, array: np.ndarray, dimensions: list[dict[str, str | float]] | None = None, attrs: dict[str, Any] = None, fig: BokehFigure | None = None, **kwargs) → BokehImage[source]ÔÉÅ

Construct a BokehImage from a 2-D numpy array

The internal data structure used is an xarray.DataArray, which is constructed by this method before initializing the class. The attributes which can be set are used by the class to define, at the present, coordinate systems for each array axis and labels (titles, units).

By default the class sets a pixel-aligned coordinate system for the primary axes ('_y', '_x'), which is used for all annotations defined on the image. To specify a secondary, dimensioned coordinate system supply the dimensions argument to this class with one of the following formats:

Dimension specification

dimension[‚Äòname‚Äô]str, required

A unique name for the dimension, used in the axis label

dimension[‚Äòflipped‚Äô]bool, optional

Whether the dimension should be flipped when displayed, False by default

dimension[‚Äòunit‚Äô]str, optional

A unit label for the dimension, applied to the axis label

dimension[‚Ä¶]str, float

See build_coord_range() for numerical dimension specification.

Similarly certain attributes can be specficied on the array:

Attribute specification

attrs[‚Äòtitle‚Äô]str, optional

A title for the array

attrs[‚Äòarray_unit‚Äô]str, optional

A unit for the array elements, appended to the title and colorbar

Parameters

  • array (np.ndarray) ‚Äì A 2-dimensional numpy array

  • dimensions (Optional[list[dict[str, Union[str, float]]]], optional) ‚Äì Coordinate system definition for each dimension of the array, by default None, in which case only the pixel coordinate system is created. If defined should be a 2-list of dicts of coordinate attributes as detailed above.

  • attrs (dict[str, Any], optional) ‚Äì Dict of extra attributes to attach to the image, by default None, as detailed above.

  • fig (Optional[BokehFigure], optional) ‚Äì An existing BokehFigure class to attach to, by default None, usually set automatically if constructed by the BokehFigure.add_image method.

  • **kwargs (dict, optional) ‚Äì Optional kwargs passed to the BokehImage init

Returns

update_raw_image(array: np.ndarray, force: bool = False, maxrate: int = 5, fix_clims: bool | tuple[float, float] = False)[source]ÔÉÅ

Update the current displayed image with the supplied array

This is the user-facing endpoint for triggering image updates

The supplied array must have the same shape as the array on which the class was initialized. If a change of shape is desired the user can try the raw_image() setter, but this is generally unsupported.

Parameters

  • array (np.ndarray) ‚Äì The array to update with

  • force (bool, optional) ‚Äì Set true to force an update to the plot even if the updates are being rate limited, otherwise, by default False

  • maxrate (int, optional) ‚Äì The maximum number of times per-second to update, by default 5. If this method is called faster than this then the internal data of the image is updated but updates are not pushed to the browser. Call refresh_display() after fast updating in order to ensure the browser version is up-to-date.

  • fix_clims (Union[bool, tuple[float, float]], optional) ‚Äì Fix the image color limits when updating, by default False. Can either be a boolean True, in which case the previous color limits are maintained, or a 2-tuple of floats which are used to force a particular color limits. This is mainly interesting when the update rate is expected to be high, as it prevents the need to calculate the limits and push updates to the browser (which then has to do the mapping). Again call refresh_display() to ensure everything is in sync after fast updating.

class aperture.display.image_datashaded.BokehImageDatashaded(*, xar: xa.DataArray, fig: BokehFigure | None = None, anchor_xy: tuple[float, float] = (0.0, 0.0), **kwargs)[source]ÔÉÅ

Bases: BokehImage

A subclass of BokehImage which automatically datashades (downsamples) large images and responsively refreshes the view as the user zooms in/out.

Uses the datashader library.

No specific parameters are required to use this class, though its‚Äô use can be enabled/disabled when constructing the BokehImage using BokehFigure.add_image() (see the signature of this method for information).

There are two class attributes to control global behaviour:

_datashade_thresholdÔÉÅ

Threshold size in bytes below which datashading will be disabled, even if this class is used. The size is chosen so as to not datashade a 1k x 1k np.float32 image which corresponds to 4 MB.

Type

float, default 1.05 * (4 * 1024**2)

_datashade_methodÔÉÅ

Method used when downsampling, supported modes are 'nearest' and 'linear'. This is a limitation of the datashader library

Type

str, default ‚Äònearest‚Äô
